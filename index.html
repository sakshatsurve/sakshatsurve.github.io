Stock Prediction using Sentimental Analysis

Details

Field: Machine Learning.
Time Span: February 2019 - May 2019.

Description

- In this project, the data is collected from Twitter and News. This data is then pre-processed, after which Semantic Analysis is done on it and a positive, negative, neutral and compound score is calculated. Then this data, with the information of stocks is passed to our training model. The model consists of LSTM (Long-Short Term Memory) and ANN(Artificial Neural Network). After training the model it is tested of the initially split test data and the results are then evaluated.
- Performed web crawling using a python script and ‘BeautifulSoup’ library to collect news from ‘businesstimes.com’ and ‘tweepy’ library to collect Twitter data. Sentiment Intensity Analyzer was used to get the negative, positive, neutral, compound scores of the content.
- The hyperparameters of the network such as weight, bias, learning rate, etc, play an important role in the prediction of the target variable, a Genetic Algorithm is induced in to the LSTM model to optimize the model and select best hyper-parameters for the given model and training dataset. Genetic Algorithm computes the fitness score of the initial random population(weights and bias) to select the hyperparameters over which crossover and mutation are performed till the population converges and we get the fittest hyperparameters.

Real Time Event Detection Using Social Sensors

Details

Field: Data Extraction
Time Span: July 2017 - March 2018.

Description

- The project investigates the real-time interaction of events such as earthquakes, in Twitter, and proposes an algorithm to monitor tweets and to detect a target event. K-means classifier is used based on features such as the keywords in a tweet, the number of words, and their context. Each Twitter user is considered as a sensor and then Kalman filtering and Particle filtering is used, which are widely used for location estimation
- By using distributing computation and data aggregation technique on the discrete signals as a new representation of the original data, we extract and track real social events on Social Data Stream. After detecting the events we notify the users using an app on the mobile which will help the nearby people to save themselves from any damage.
- To detect the authenticity of the tweets we have planned to consider the frequency of tweets. Additionally, we also take into consideration the number of tweets coming from a particular area, typically from the area affected from event or the areas in the vicinity.
- Firebase Cloud Messaging (FCM) is used to notify users in real-time via a text message. FCM offers a broad range of messaging options and capabilities.

Implementation of GET and PUT HTTP commands using Multithreading

Details

Field: Computer Networks.
Time Span: March 2019.

Description

- A client sends an HTTP request to a server in the form of a request message, with a header and a message body containing the payload. Then, a server responds to a client's request by sending one or more HTTP response messages
- The server is able to handle multiple simultaneous service requests in parallel. This means that the Web server is multi-threaded. A threaded web server is one that handles each request with a new thread, as opposed to handling each request with a new process. Multi-threading is more complex than concurrency with processes, so there's an increased risk of the web server crashing if one of its modules isn't working correctly.
- The project was implemented on Python and initiated by using Command Line arguments.
